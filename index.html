Sobre o Projeto

Este projeto consiste em uma análise crítica do site institucional da Universidade Federal de Santa Catarina (UFSC), identificando problemas reais de usabilidade, acessibilidade e interação. Cada problema identificado é fundamentado teoricamente com base nas heurísticas de Nielsen e princípios de IHC, e soluções práticas são propostas através de um protótipo mockado.

O objetivo é demonstrar como princípios teóricos de IHC podem ser aplicados na prática para melhorar a experiência do usuário em sistemas reais, consolidando o conhecimento adquirido na disciplina de Interface Humano-Computador.

Problemas Identificados e Correções

Análise comparativa entre o site original ufsc.br e a versão corrigida, fundamentada nas 10 Heurísticas de Usabilidade de Jakob Nielsen. Cada problema apresenta comparação visual e lista detalhada das correções implementadas.

1

Navegação Complexa e Sobrecarga Cognitiva

#6 #8 #3

❌ Site Original

  • Menu com 50+ links simultâneos
  • Múltiplos níveis hierárquicos confusos
  • Sobrecarga cognitiva para usuários
  • Dificuldade em localizar informações

✅ Versão Corrigida

  • Menu simplificado (6 itens principais)
  • Seção "Acesso Rápido" com cards visuais
  • Busca destacada e acessível
  • Navegação orientada a tarefas do usuário

Correções Implementadas:

  • ✓ Redução do menu principal de 50+ para 6 itens principais
  • ✓ Criação de seção "Acesso Rápido" com 8 serviços mais utilizados
  • ✓ Cards visuais com ícones para identificação rápida
  • ✓ Busca destacada na navegação principal
  • ✓ Estrutura hierárquica clara e intuitiva

Fundamentação: Nielsen (1994) - Interfaces devem minimizar carga cognitiva, apresentando informações de forma clara e hierárquica. Norman (2013) enfatiza que navegação deve seguir modelos mentais intuitivos.

Falta de Feedback Visual em Interações

#1 #4 #7

❌ Site Original

  • Links sem estados hover claros
  • Botões sem feedback visual
  • Elementos não indicam interatividade
  • Incerteza sobre ações do sistema

✅ Versão Corrigida

  • Estados hover/focus/active em todos elementos
  • Transições CSS suaves
  • Indicadores de foco visíveis
  • Carrossel com controles e indicadores ativos

Correções Implementadas:

  • ✓ Estados hover implementados em links e botões
  • ✓ Transições CSS suaves para feedback visual
  • ✓ Indicadores de foco visíveis para navegação por teclado
  • ✓ Carrossel com indicadores ativos e controles visíveis
  • ✓ Efeitos de elevação em cards ao passar mouse

Fundamentação: Nielsen (1994) - Sistema deve sempre informar ao usuário o que está acontecendo através de feedback apropriado. Norman (2013) enfatiza importância de affordances visuais claras que indiquem como elementos podem ser manipulados.

Hierarquia Visual Inconsistente

#8 #6 #4

❌ Site Original

  • Múltiplos elementos competindo pela atenção
  • Sem hierarquia visual clara
  • Títulos com tamanhos similares
  • Confusão sobre o que é mais importante

✅ Versão Corrigida

  • Banner hero dominante no topo
  • Sistema de tipografia hierárquico claro
  • Grid system consistente
  • Contraste e espaçamento adequados

Correções Implementadas:

  • ✓ Carrossel hero com título grande e destaque visual
  • ✓ Sistema de tipografia claro (H1-H6 com tamanhos distintos)
  • ✓ Grid system consistente para organização
  • ✓ Contraste e espaçamento adequados entre seções
  • ✓ Elementos importantes destacados com cores e tamanhos

Fundamentação: Preece et al. (2015) - Hierarquia visual guia atenção e facilita compreensão. O uso adequado de tipografia, espaçamento e contraste cria uma "arquitetura visual" que orienta o usuário.

Linguagem Técnica Não Adequada

#2 #6 #10

❌ Site Original

  • Siglas sem contexto (PROGRAD, PROEX, CAGR)
  • Jargão acadêmico sem explicações
  • Terminologia técnica assumindo conhecimento prévio
  • Barreira de compreensão para novos usuários

✅ Versão Corrigida

  • Linguagem orientada a tarefas do usuário
  • Nomes descritivos completos
  • Textos introdutórios em linguagem clara
  • Eliminação de siglas desnecessárias

Correções Implementadas:

  • ✓ Menu principal usa linguagem do usuário ("Ensino", "Pesquisa", "Extensão")
  • ✓ Seção "Acesso Rápido" com nomes descritivos completos
  • ✓ Textos introdutórios em linguagem clara e acessível
  • ✓ Eliminação de siglas desnecessárias no menu principal
  • ✓ Contexto fornecido para termos técnicos quando necessário

Fundamentação: Norman (2013) - Sistemas devem usar linguagem do mundo real, não jargão técnico. Nielsen (1994) destaca que interface deve falar a linguagem do usuário, seguindo convenções do mundo real.

Inconsistência Visual e de Padrões

#4 #6 #3

❌ Site Original

  • Estilos diferentes entre páginas
  • Espaçamentos variáveis
  • Cores não seguem padrão consistente
  • Elementos similares se comportam diferente

✅ Versão Corrigida

  • Design system unificado
  • Componentes padronizados
  • Paleta de cores consistente
  • Espaçamentos padronizados

Correções Implementadas:

  • ✓ Design system centralizado (assets/css/style.css)
  • ✓ Paleta de cores consistente (azul UFSC #003366)
  • ✓ Componentes reutilizáveis (botões, cards, formulários)
  • ✓ Espaçamentos padronizados (variáveis CSS)
  • ✓ Tipografia consistente em toda a interface

Fundamentação: Nielsen (1994) - Usuários não devem se perguntar se palavras, situações ou ações diferentes significam a mesma coisa. Consistência reduz carga cognitiva e aumenta eficiência. Norman (2013) destaca que padrões consistentes criam affordances previsíveis.
